# 总体进度

## 使用说明

1. 每个模块对应一个独立任务文件。
2. 子任务采用 checklist 形式管理。
3. 完成后将 `- [ ]` 改为 `- [x]`。
4. 建议按“基础设施 -> 规则与数据 -> 业务服务 -> API -> 前端”顺序推进。
